# 自定义bootstrap sass变量

Bootstrap版本： v4.5.3 在自定义bootstrap全局的sass变量之前，需要在vue项目中配置vue.config.js，暴露全局的sass变量，让这些变量在整个vue项目中的style代码块中全局可用，假设我们使用custom.scss来引用和新增bootstrap sass全局变量，则配置如下：

    module.exports = {
    css: {
        loaderOptions: {
        scss: {
            prependData: `@import "@/common/sass/theme/custom.scss";`
        }
        }
    }
}

如果需要全局暴露的scss文件不止一个，则设置prependData属性为以";"分隔的多个@import语句：

    prependData: `@import "@/common/sass/global.scss";@import "@/common/sass/theme/custom.scss";`

通过npm install bootstrap -S安装bootstrap后，它的全局sass变量定义在/node_modules/bootstrap/scss/_variables目录中，此目录里的文件使用了bootstrap定义的全局sass函数，所以，在覆盖bootstrap定义的全局变量时，需要导入这两个目录下的文件；在custom.scss文件中写入如下代码：

/* begin */                                                 
// 在这里定义自己的bootstrap变量，覆盖默认bootstrap的值 
$white: purple;

/* end */
@import "../../../../node_modules/bootstrap/scss/functions";
@import "../../../../node_modules/bootstrap/scss/_variables";

// 新增全局scss变量
// $xs-bks
$xs-bks: nth(nth($grid-breakpoints, 1), 2); 
// $sm-bks
$sm-bks: nth(nth($grid-breakpoints, 2), 2);
// $md-bks
$md-bks: nth(nth($grid-breakpoints, 3), 2);
// $lg-bks
$lg-bks: nth(nth($grid-breakpoints, 4), 2);
// $xl-bks
$xl-bks: nth(nth($grid-breakpoints, 5), 2);

以上代码，我们覆盖了bootstrap的全局sass变量$white(覆盖sass变量，一定要在导入bootstrap的sass变量之前定义好需要覆盖的变量);

同时新增了五个全局变量：$xs-bks、$sm-bks、$md-bks、$lg-bks、$xl-bks；这几个变量都是从全局变量$grid-breakpoints中获取的；$grid-breakpoints在sass变量类型中是一个数组：

    $grid-breakpoints: (
        xs: 0,
        sm: 576px,
        md: 768px,
        lg: 992px,
        xl: 1200px
    ) !default;

!default表示是默认值，如果在此之前，已经定义过$grid-breakpoints，则以之前定义的值为准

从数组中取值

    nth($grid-breakpoints, 2) // 从数组$grid-breakpoints中取第二个值，也就是"sm: 567px"

上面取出的值"sm: 576px"，在sass中也属于数组，再使用nth((sm: 576px), 2)就能取出576px了; 因此，新增的五个全局变量的值分别是： $xs-bks: 0
$sm-bks: 576px
$md-bks: 768px
$lg-bks: 992px
$xl-bks: 1200px
